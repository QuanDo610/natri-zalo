🔑 KEY FEATURES
═══════════════════════════════════════════════════════════════════════════════

✨ Features Implemented:

1. PERMISSION WORKFLOW
   • checkZaloCameraPermission() - check status
   • requestCameraPermission() - request permission
   • Automatic fallback if denied
   • Single permission prompt (not repeated)

2. HYBRID CAMERA SELECTION
   • Zalo Mini App API (first choice):
     ✓ Better performance
     ✓ Native device integration
     ✓ Better battery efficiency
   
   • getUserMedia fallback (always-on):
     ✓ Works on web browsers
     ✓ HTTPS required (except localhost)
     ✓ Same barcode detection
     ✓ No degraded experience

3. REALTIME BARCODE SCANNING
   • ROI-based detection (70% center region)
   • Frame upscaling (1.8x for clarity)
   • Multiple enhancement strategies:
     ✓ Grayscale
     ✓ Contrast
     ✓ Sharpen
     ✓ Binary
     ✓ Extreme
   • ZXing/Quagga decode with TRY_HARDER
   • Fast detection: ~400-600ms typical

4. HIGH-QUALITY CAPTURE
   • Freeze ROI canvas at detection moment
   • Save as JPEG (quality 0.95)
   • Full resolution barcode (not downsampled)
   • User sees exactly what was scanned

5. OPTIONAL FULL PHOTO (Zalo only)
   • cameraContext.takePhoto() integration
   • Separate from ROI capture
   • Not used for barcode decode
   • User can save full context if desired
   • No camera restart needed

6. CAMERA SWITCHING
   • getCameraList() - enumerate available cameras
   • setDeviceId() / switchCamera() - switch devices
   • UI dropdown for camera selection
   • Instant switch (Zalo) vs restart (getUserMedia)
   • Helpful for multi-camera phones

7. NO CAMERA RESTART AFTER CAPTURE
   • Current code: Camera kept running after capture
   • Old x flow: Stop → Restart → User adjusts position
   • New flow: ✓ Immediate re-scan or upload
   • Better UX, faster workflow

8. TYPE-SAFE IMPLEMENTATION
   • Full TypeScript interfaces for all Zalo APIs
   • No @types packages needed
   • Compile-time error detection
   • Inline documentation

    │   │   └─ No permission: Request → Grant/Deny
    │   │                     if Grant: Create stream → Success ✓
    │   │                     if Deny: Continue to fallback
    │   └─ No: Continue to fallback
    │

            ├─ Try 4:3 constraints
            │
            ├─ If failed: Try 16:9 constraints
            │   │
            │   └─ If success: Stream created ✓
            │
            └─ If all failed: Show error
    ↓

    ├─ Poll video element every 400ms
    ├─ Extract ROI region
    ├─ Apply enhancement
    ├─ Attempt decode (ZXing + Quagga)
    │   ├─ Success: Freeze canvas to blob → Show barcode ✓
    │   └─ Failed: Try next enhancement
    └─ Loop until detection or user stops

📞 SUPPORT & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Common Issues:

1. Zalo camera not detected
   → Check if running on Zalo mini app (not browser)
   → Verify zmp-sdk loaded: window.zalo?.camera
   → Call: ZaloCamera.isZaloCameraAvailable()

2. Permission request not showing
   → Permission may already be granted
   → Check: await ZaloCamera.checkPermission()
   → Try: Zalo app > Settings > Permissions

3. Blank screen after camera opens
   → Wait 2-3 seconds for focus to settle
   → Check video element CSS: objectFit should be 'contain'
   → Check console for errors

4. Barcode not detected
   → Move barcode to center of ROI frame
   → Check lighting (too dark = no detection)
   → Try different barcode angle
   → Zoom in if barcode is small
   → Check browser console for enhancement strategies

5. Full photo not working
   → Only available with Zalo camera
   → Check: usingZaloCamera flag = true
   → Some Zalo devices may not support takePhoto()

For detailed troubleshooting: See ZALO_CAMERA_INTEGRATION_GUIDE.ts
